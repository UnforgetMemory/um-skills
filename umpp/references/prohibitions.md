# 🚫 禁止行为清单

违反以下任何一条均为 **严重违规**，需要在 Review 阶段被拦截。

## 架构与设计

| # | 禁止行为 | 原因 |
|---|---------|------|
| 1 | 大规模重构替代局部修复 | 引入不必要风险，扩大变更范围 |
| 2 | 创建空抽象层（Interface 无多实现） | 违反 YAGNI，增加认知负载 |
| 3 | 引入无必要依赖 | 增加构建时间与维护成本 |
| 4 | 修改无关模块 | 超出 Scope，增加 Review 负担 |
| 5 | 为不存在未来创建泛化设计 | 违反 Evolution First 原则 |

## 执行与验证

| # | 禁止行为 | 原因 |
|---|---------|------|
| 6 | 未验证直接宣称「完成」 | 违反 Phase 5 要求 |
| 7 | 跳过 Review 阶段 | 质量门禁不可绕过 |
| 8 | 压低测试标准绕过 CI | 测试是安全网，不是障碍 |
| 9 | 跳过回归验证 | 功能性/性能回归必须检验 |

## 协作与变更

| # | 禁止行为 | 原因 |
|---|---------|------|
| 10 | 自动 git commit | 变更必须人工审查 |
| 11 | 自动 git push | 远程变更必须人工确认 |
| 12 | 自动 release / 版本发布 | 发布流程必须人工触发 |
| 13 | 多个 Agent 同时修改同一文件 | 导致冲突，违反协作纪律 |
| 14 | 提交包含调试代码 / TODO / 死代码 | 工程卫生不合格 |

## 信息处理

| # | 禁止行为 | 原因 |
|---|---------|------|
| 15 | 猜测 API 行为 | 必须查官方文档 |
| 16 | 基于无依据经验做假设 | 必须标记为 `Assumption` |
| 17 | 伪造不存在的 API 或参数 | 数据造假，不可接受 |
| 18 | 使用非权威/过时来源 | 优先官方文档 |
